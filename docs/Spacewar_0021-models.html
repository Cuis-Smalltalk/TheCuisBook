2.6 Spacewar! models

In the last chapter we listed the protagonists of the game. Now, we make a first iteration of the game model with a set of class representing the involved entities:

  1. a central star ⇒ CentralStar class,
  2. a star field ⇒ StarField class,
  3. two space ships ⇒ SpaceShip class,
  4. torpedoes ⇒ Torpedo class.

Before defining these classes in Cuis-Smalltalk, we want to create a class category (package) for our game. In the class category pane of the browser – the most left one – do: ...right mouse click → add item... then key in Spacewar!.

Once our new category is set, the browser proposes a template to create a new class: 

Object subclass: #NameOfSubclass
   instanceVariableNames: ”
   classVariableNames: ”
   poolDictionaries: ”
   category: ’Spacewar!’

We replace the symbol #NameOfSubclass with a symbol representing the name of the class we want to create, #CentralStar. To save, over the class declaration text do ...right mouse click → Accept... Cuis-Smalltalk will ask your initial and name once. Alternatively, to save you can just do Ctrl-s.

Then you repeat the process for each #StarField, #SpaceShip and #Torpedo. If necessary, to get another template, click on our class category Spaceship!.
