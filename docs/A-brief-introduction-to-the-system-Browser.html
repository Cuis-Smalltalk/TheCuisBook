2.5 A brief introduction to the system Browser

The System Browser, in short the Browser, is a tool to rule all the classes in Cuis-Smalltalk. It is both a tool to explore the classes (system or user ones) and to write new classes and methods.

To access the tool, ...Background click → Open... → Browser...

Figure 2.1: The System Browser

At the top left are the class categories, there are group of classes part of the same theme. It is also called Package, it is an organisational element to save code. In Figure 2.1, the selected class category is Kernel-Numbers, a group of classes we already explored. The term Kernel- indicates it is part of fundamental categories, but it is only a convention. See the other categories as Kernel-Text and Kernel-Chronology related to text and date entities.

Next to the right are the classes in the selected class category. They are nicely presented in a parent-child class hierarchy. When a class is first selected in this pane, its declaration appears in the large pane bellow, the Number class declaration is: 

Magnitude subclass: #Number
   instanceVariableNames: ”
   classVariableNames: ”
   poolDictionaries: ”
   category: ’Kernel-Numbers’

Observe a few important points in this declaration:

  • Number is a sub class of Magnitude. This means Number is a kind of specialized Magnitude.
  • the declaration itself is Smalltalk code, indeed the message #subclass:instanceVariableNames:classVariableNames:poolDictionaries: category: is sent to Magnitude
  • the subclass: argument Number is prefixed with “#”, it is a symbol, a kind of unique string. Indeed when declaring the Number class, the system does not know it yet, so it is named as a symbol.
  • The instanceVariableNames: argument is a string: the instance variables of the class are declared by name separated by a space. There is none in this class definition.

To learn about the intend of a class, you must visit systematically its comment. Often a comment comes also with code examples to learn how to use the object; these code snippets can be selected and executed in place as done from a Workspace. In Figure 2.1, see the comment button to read or to edit the comment of the selected class.

Next to the right is the method categories panel. It is user defined method categories. A class can come with tenth of methods, grouping them by category ease the user access. As a reference, Number has more than 100 instance side methods implemented in itself7; therefore clicking the arithmetic category directly gives access to related methods in the next and last pane at the right.

In the Browser, once a method is selected – in Figure 2.1, abs method – the bottom part shows its source code, ready to be explored or edited. Again, often, you will read a small comment just after the method name, it is surrounded by double quote.

A last bit of advice, in the workspace when requesting the class of an instance, Ctrl-b will invoke the Browser on the returned class:

  • In the Workspace, on 1 class execute and print with Ctrl-p,
  • SmallInteger is printed and automatically selected by default,
  • Invoke the Browser on the selected SmalltalkInteger class with Ctrl-b,
  • The Browser is open on the SmalltalkInteger, ready to be explored.
